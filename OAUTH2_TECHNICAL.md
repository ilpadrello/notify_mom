# OAuth2 Technical Implementation Details

## Architecture

### OAuth2 Manager (`src/oauth.ts`)

Core OAuth2 functionality:

```typescript
class OAuth2Manager {
  - oauth2Client: OAuth2Client

  Methods:
  - hasValidToken(): boolean
  - loadTokenFromFile(): boolean
  - getAuthUrl(): string
  - getAccessToken(code: string): Promise<void>
  - getClient(): OAuth2Client
  - refreshToken(): Promise<void>
  - printAuthInstructions(): void
}
```

**Key Functions:**

- **hasValidToken()** - Checks if `token.json` exists with valid access token
- **loadTokenFromFile()** - Loads token from `token.json` and sets it in OAuth2 client
- **getAuthUrl()** - Generates Google login URL with read-only Sheets scope
- **getAccessToken(code)** - Exchanges authorization code for tokens (saves to `token.json`)
- **refreshToken()** - Refreshes expired token and updates `token.json`

### Authentication CLI (`src/auth.ts`)

User-facing authentication tool:

```bash
npm run auth              # Prints instructions + auth URL
npm run auth -- CODE     # Exchanges code for token
```

**Process:**

1. User runs `npm run auth`
2. OAuth2Manager generates auth URL
3. User visits URL and authorizes
4. Browser redirects with authorization code
5. User copies code and runs `npm run auth -- CODE`
6. Auth.ts exchanges code for tokens
7. Tokens saved to `token.json`

### Token File (`token.json`)

Structure:

```json
{
  "type": "authorized_user",
  "client_id": "...",
  "client_secret": "...",
  "refresh_token": "...",
  "access_token": "...",
  "access_token_expiry": 1234567890000,
  "scope": "https://www.googleapis.com/auth/spreadsheets.readonly"
}
```

**Generated by:**

- `npm run auth` (initial authentication)

**Used by:**

- OAuth2Manager.loadTokenFromFile()
- Google Sheets sync engine

**Security:**

- ✅ In `.gitignore` (won't be committed)
- ✅ Contains refresh token (long-lived)
- ✅ Contains access token (auto-refreshed)
- ⚠️ Keep private! Don't share this file!

## OAuth2 Flow

### 1. Initial Authentication

```
User                    App                    Google
  │                     │                        │
  ├─── npm run auth ────→│                        │
  │                     │                        │
  │                     ├── Generate Auth URL ──→│
  │                     │                        │
  │ ← Print Instructions │                        │
  │ + URL               │                        │
  │                     │                        │
  ├─ Click URL ────────────────────────────────→│
  │                     │                        │
  │ ← Sign In ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ← ←│
  │                     │                        │
  ├─ Authorize ───────────────────────────────→│
  │                     │                        │
  │ ← Redirect with code │                        │
  │                     │                        │
  │ Copy code           │                        │
  │                     │                        │
  ├─ npm run auth CODE ─→│                        │
  │                     ├── Exchange Code ──────→│
  │                     │                        │
  │                     │←─ Access Token ────────│
  │                     │                        │
  │                     ├─ Save token.json       │
  │                     │                        │
  │ ← Success           │                        │
  │                     │                        │
```

### 2. Application Runtime

```
npm start
    │
    ├─ Load .env
    ├─ Check token.json exists?
    │
    ├─ If valid:
    │   ├─ Load token into OAuth2Client
    │   ├─ Connect to Google Sheets
    │   └─ Begin syncing
    │
    └─ If invalid:
        └─ Error: "No valid token. Run npm run auth"
```

### 3. Token Refresh

```
During sync/operation:
    │
    ├─ Check token expiry
    │
    ├─ If expired:
    │   ├─ Request new access token (using refresh token)
    │   ├─ Update token.json
    │   └─ Continue operation
    │
    └─ If valid:
        └─ Use token as-is
```

## Security Considerations

### Tokens

✅ **Good:**

- Access tokens are short-lived (1 hour)
- Refresh tokens are long-lived (offline access)
- Tokens stored only locally in `token.json`
- Tokens in `.gitignore` (won't be shared via git)
- Read-only scope (can't modify sheets)

⚠️ **Caution:**

- Don't commit `token.json` to git
- Don't share `token.json` with others
- Keep `.env` with Client Secret secure
- Revoke access if device compromised

### Scopes

```
https://www.googleapis.com/auth/spreadsheets.readonly
```

**Grants:**

- ✅ Read access to spreadsheets
- ❌ Cannot modify spreadsheets
- ❌ Cannot delete spreadsheets
- ❌ Cannot access other APIs

### Revocation

Users can revoke access:

1. Visit: https://myaccount.google.com/permissions
2. Find "Notify Mom"
3. Click "REMOVE ACCESS"

Immediate effect:

- App can no longer access sheets
- Need to re-authenticate: `npm run auth`

## Comparison with Alternatives

### OAuth2 (Current)

```
Pros:
- ✅ User account (acts as you)
- ✅ Secure tokens
- ✅ Automatic refresh
- ✅ No credential files to manage
- ✅ Can revoke anytime

Cons:
- ⚠️ Requires initial interactive auth
- ⚠️ Tokens per machine/device
```

### Service Account (Previous)

```
Pros:
- ✅ No interactive auth
- ✅ Works everywhere with same token

Cons:
- ❌ Separate service account
- ❌ JSON credential file to manage
- ❌ Less secure for personal use
- ❌ Credentials shared if file leaked
```

### API Key (Not Used)

```
Pros:
- ✅ Simple

Cons:
- ❌ Public sheets only
- ❌ No write access
- ❌ Rate limited
```

## Error Handling

### Common Errors

**1. No token.json**

```
Error: No valid authentication token found
Fix: npm run auth
```

**2. Invalid credentials**

```
Error: OAuth2 credentials not configured
Fix: Check .env has GOOGLE_OAUTH_CLIENT_ID and SECRET
```

**3. Token expired**

```
Handled automatically by: oauth2Manager.refreshToken()
No user action needed
```

**4. Authorization failed**

```
Error: Unable to exchange authorization code
Fix: Code might have expired, retry npm run auth
```

**5. Spreadsheet not found**

```
Error: Failed to load Google Sheet
Fix: Check GOOGLE_SHEETS_SPREADSHEET_ID is correct
```

## Code Example

### Using OAuth2Manager

```typescript
import oauth2Manager from "./oauth.js";

// Check if authenticated
if (!oauth2Manager.hasValidToken()) {
  console.log("Not authenticated yet!");
  console.log(oauth2Manager.getAuthUrl());
  process.exit(1);
}

// Load token
if (!oauth2Manager.loadTokenFromFile()) {
  console.error("Failed to load token");
  process.exit(1);
}

// Use with google-spreadsheet
import { GoogleSpreadsheet } from "google-spreadsheet";

const doc = new GoogleSpreadsheet(spreadsheetId, oauth2Manager.getClient());

await doc.loadInfo();
console.log("Connected:", doc.title);
```

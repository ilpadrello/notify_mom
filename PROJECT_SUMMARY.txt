DATABASE SCHEMA
================================================================================
Table: schedule_entries
  id TEXT PRIMARY KEY       -- YYYY-MM-DD_HH:MM
  month TEXT NOT NULL       -- Italian month name
  day INTEGER NOT NULL      -- 1-31
  time TEXT NOT NULL        -- HH:MM format
  name TEXT NOT NULL        -- Entry name (e.g., "Maria")
  created_at TEXT NOT NULL  -- ISO 8601 timestamp
  updated_at TEXT NOT NULL  -- ISO 8601 timestamp
  Indices: month_day, time

Table: sync_logs
  id INTEGER PRIMARY KEY AUTOINCREMENT
  sync_timestamp TEXT       -- Operation timestamp
  action TEXT               -- INSERT, UPDATE, DELETE
  entry_id TEXT            -- Foreign key reference
  details TEXT             -- JSON serialized data
  created_at TEXT NOT NULL -- ISO 8601 timestamp
  Indices: sync_timestamp, action

SYNC FLOW
================================================================================
1. Scheduler triggers every 15 minutes (configurable)
2. Sync Engine fetches data from Google Sheets API
3. Parser extracts columns: Mese, Date, Time, Nonna
4. Filter applied: Keep only "maria" entries (case-insensitive)
5. Unique IDs generated: YYYY-MM-DD_HH:MM
6. Database comparison: Identify added/removed entries
7. Operations executed:
   - New entries → INSERT to DB + NOTIFY_NEW
   - Removed entries → DELETE from DB + NOTIFY_ANNULLED
8. Notifications sent in parallel (Email + Telegram)
9. Sync logged to sync_logs table
10. Process repeats in 15 minutes

INSTALLATION INSTRUCTIONS
================================================================================
1. Install dependencies:
   $ npm install

2. Configure environment:
   $ cp .env.example .env
   # Edit .env with your Google Sheets & notification credentials

3. Build TypeScript:
   $ npm run build

4. Run application:
   $ npm start          # Production
   $ npm run dev        # Development (auto-reload)

5. (Optional) Deploy with Docker:
   $ docker-compose up -d
